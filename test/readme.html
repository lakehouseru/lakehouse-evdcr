ВАЖНО: прочитайте это перед установкой

Чтобы не получить ошибку вида CANNOT MODIFY HEADER INFORMATION, не пользуйтесь стандартным БЛОКНОТОМ (для пользователей ОС Windows) для редактирования каких-либо файлов WordPress.
Особенно это касается файла wp-config.php
В WordPress используется кодировка utf-8
Если вы вносите изменения в файлы движка или файлы Темы, необходимо сохранять их в кодировке utf-8 без сигнатуры BOM
НЕ ВСЕ РЕДАКТОРЫ УМЕЮТ РАБОТАТЬ С UTF-8 БЕЗ BOM

Отключайте BOM
Воспользуйтесь лучше одним из этих редакторов:
UnicEdit,
Notepad++,
Notepad2.
При сохранении файла многие текстовые редакторы предлагают флажок «Include Unicode Signature (BOM)», «Add Byte Order Mark» или нечто подобное. Прежде всего убедитесь, что в вашем редакторе это есть. Если похожей настройки не обнаружено (как, например, в «Блокноте») — пользоваться таким редактором для серьёзных задач не стóит. Найдя этот флажок — отключите его.
Рекомендуется также почитать эту запись и пользоваться плагином поиска BOM в будущем.

Установка: Знаменитая установка за 5 минут

  1. Распакуйте архив в пустую директорию.
  2. Откройте wp-config-sample.php текстовым редактором, например WordPad или аналогичным и заполните данные для подключения к базе данных.
  3. Сохраните файл как wp-config.php
  4. Загрузите все на сервер.
  5. Откройте /wp-admin/install.php в вашем браузере. Это должно настроить таблицы, необходимые для блога. Если будет ошибка, то дважды проверьте ваш файл wp-config.php и попробуйте заново. Если ошибка произойдет снова, пожалуйста обратитесь в  форум поддержки (англ) или русский форум поддержки и соберите как можно больше данных, которые смогут пригодиться чтобы вам помочь.
  6. Пароль дается вам при установке.
  7. Скрипт установки затем должен отправить вас на страницу входа. Войдите с именем пользователя admin и паролем, сгенерированном для вас во время установки. Вы можете затем перейти в раздел 'Профиль' чтобы изменить пароль.

Обновление

Используя автоматическое обновление

Если у вас установлен Wordpress 2.7 или новее, вы можете воспользоваться автоматическим обновлением:

  1. Откройте ссылку wp-admin/update-core.php в вашем браузере и следуйте инструкции.
  2. Ожидали большего? Это все!

Обновление вручную

  1. Перед тем как приступить к обновлению, СДЕЛАЙТЕ РЕЗЕРВНЫЕ КОПИИ всех важных файлов, которые вы изменяли,  например index.php.
  2. Удалите файлы прежней версии WordPress, сохраните те, что вы изменили.
  3. Загрузите новые файлы.
  4. Откройте в бразуере ссылку /wp-admin/upgrade.php.

Переход с других систем

WordPress может импортировать данные с различных систем. Но прежде вам нужно установить и настроить наши инструменты импорта.

Системные требования

  • PHP версии 4.3 или выше.
  • MySQL версии 4.1.2 или выше.

Системные рекомендации

  • Модуль Apache mod_rewrite.
  • Ссылка на http://wordpress.org с вашего сайта.

Ресурсы в интернете

Если у вас имеются вопросы, не касающиеся этого документа, воспользуйтесь нашими сетевыми ресурсами:

Кодекс WordPress
Кодекс — это энциклопедия обо всем, что касается WordPress. Это наиболее полный источник информации о WordPress. (Не вся информация представлена на русском языке, но вы можете помочь локализовать некоторые материалы)
Блог WordPress
Здесь вы можете найти информацию о последних обновлениях и новости авторов WordPress. Свежие новости о WordPress по умолчанию появляются в вашей панели администрирования.
Планета WordPress
Планета WordPress — это агрегатор новостей с блогов, посвящённых WordPress, по всему миру (в основном информация на английском языке).
Форумы поддержки WordPress 
Если вы уже смотрели везде и не нашли ответа, на форумах поддержки есть большое и активное сообщество людей, готовых оказать помощь. Чтобы вам помогли и не отправили "в поиск", используйте информативный заголовок сообщения и сформулируйте свой вопрос максимально детально.
WordPress IRC канал
Существует канал онлайн-чата, который используется для обсуждения среди людей, использующих WordPress, и иногда для поддержки. Страница по ссылке выше поможет вам попасть туда. (irc.freenode.net #wordpress)

XML-RPC и Atom интерфейс

Вы можете публиковать в ваш блог WordPress с помощью инструментов Windows Live Writer, Ecto, w.bloggar, Radio Userland (это означает, можно использовать функцию Radio email-to-blog), NewzCrawler, и другие инструменты, которые поддерживают Blogging API! :) Вы можете прочитать больше о XML-RPC support on the Codex.

Публикация через электронную почту

Вы можете публиковать в блог через почтовый клиент! Для установки перейдите в экран настроек "Написание" и заполните настройки подключения для вашего секретного аккаунта POP3. Затем вы должны настроить wp-mail.php выполняться периодически для проверки новых записей. Вы можете сделать это с помощью Cron-jobs, или если ваш хостинг не поддерживает это, вы можете присмотреться к некоторым сервисам мониторинга сайтов, и заставить их проверять ваш wp-mail.php URL.

Публикация очень простая: Любое электронное письмо, отправленное на адрес будет автоматически опубликовано, с темой письма в качестве заголовка. Лучше держать этот адрес в секрете. Скрипт удаляет электронные письма которые были опубликованы.

Последние примечания

  • Если у вас есть предложение, идеи, замечания или (вдруг!) вы обнаружили ошибку, добро пожаловать в форумы поддержки (англ).
